我现在做客户模型在 DCU 上的适配和性能优化，所以学 CV 不能只知道论文名字。必须能看懂客户模型属于哪类任务：分类、检测、分割、关键点、视觉 Transformer、视觉语言、智驾感知；也要知道它为什么慢：卷积、attention、多尺度特征、RoIAlign、NMS、上采样、后处理，还是数据预处理。

这份清单的标准是：足够主流，足够经典；不了解这些，就很难说自己真懂 CV 工程优化。

## 先建立 CV 工程地图

CV 任务常见几类：

- 分类：整张图输出类别，核心是 backbone 表征能力。
- 检测：输出 box + class，核心是多尺度目标定位和后处理。
- 语义分割：每个像素分类，核心是高分辨率特征和上采样。
- 实例分割：每个物体一个 mask，核心是检测 + mask head。
- 姿态/关键点：输出点位置，核心是 heatmap 或 coordinate regression。
- 视觉语言：图像和文本对齐，核心是 image encoder + text encoder/LLM。
- 智驾感知：2D/3D 检测、BEV、occupancy、map、时序融合，单独有一套概念。

从 DCU 优化视角，CV 模型常见瓶颈：

- CNN backbone：conv、BN、activation、feature pyramid。
- Transformer backbone：attention、MLP、LayerNorm、patch embedding。
- 检测 head：大量小 tensor、多尺度 head、decode、NMS。
- 分割 head：高分辨率 feature、upsample、concat、deconv、mask loss。
- 多尺度/多分支：shape 多、kernel 小、调度开销高。
- 数据预处理：resize、decode、normalize、letterbox，有时端到端瓶颈不在模型。

## 1. AlexNet

论文：ImageNet Classification with Deep Convolutional Neural Networks, 2012。链接：https://papers.nips.cc/paper/4824-imagenet-classification-with-deep-convolutional-neural-networks

为什么必须懂：现代深度学习 CV 的起点。它证明 CNN + GPU + 大规模数据能远超手工特征。

核心机制：卷积提局部模式，ReLU 改善梯度，pooling 降采样，dropout 抑制过拟合，数据增强提升泛化。

工程含义：CV 一开始就是硬件驱动的领域。GPU/DCU 适配不是附属工作，而是模型能力释放的一部分。

## 2. VGG

论文：Very Deep Convolutional Networks for Large-Scale Image Recognition, 2014。链接：https://arxiv.org/abs/1409.1556

为什么必须懂：VGG 用规则的 3x3 卷积堆叠出深网络，结构简单，长期作为 backbone 和教学基线。

核心机制：多个 3x3 卷积叠加扩大感受野，同时增加非线性表达。

工程含义：VGG 这类网络 conv 密集，结构规则，适合理解卷积算子吞吐。但它参数和计算都重，现代模型常用 ResNet/ConvNeXt/ViT 替代。

## 3. ResNet

论文：Deep Residual Learning for Image Recognition, 2015。链接：https://arxiv.org/abs/1512.03385

为什么绝对经典：ResNet 是 CNN backbone 的分水岭。大量检测、分割、OCR、工业视觉、医学视觉模型仍以 ResNet 或其变体为骨干。

核心机制：残差连接让网络学习 F(x)+x，改善深层网络优化。bottleneck block 用 1x1 降维/升维，3x3 做空间卷积。

工程含义：ResNet 性能主要看 conv、BN、ReLU、add。推理部署常做 BN folding、conv+activation fusion、layout 优化。训练时 BN 的 batch 行为、SyncBN、混合精度稳定性也要关注。

## 4. MobileNet / EfficientNet

代表：MobileNetV1/V2/V3，EfficientNet。链接：https://arxiv.org/abs/1704.04861、https://arxiv.org/abs/1905.11946

为什么必须懂：客户边缘部署和实时推理经常用轻量 CNN。轻量模型不一定更容易优化，因为 depthwise conv 常常算力低、访存占比高。

核心机制：MobileNet 用 depthwise separable convolution 降低 FLOPs；EfficientNet 用 compound scaling 平衡 depth、width、resolution。

工程含义：Depthwise conv 的理论 FLOPs 很小，但硬件利用率可能差。DCU 上如果客户模型大量 depthwise、group conv、小 batch，可能需要专门 kernel 或图融合，否则看起来 FLOPs 低但跑不快。

## 5. ConvNeXt

论文：A ConvNet for the 2020s, 2022。链接：https://arxiv.org/abs/2201.03545

为什么必须懂：ConvNeXt 说明 CNN 不是过时路线。通过借鉴 ViT 训练和结构设计，现代 CNN 仍能作为强 backbone。

核心机制：大 kernel depthwise conv、inverted bottleneck、LayerNorm、GELU、现代训练 recipe。

工程含义：ConvNeXt 把 CNN 和 Transformer 的工程特征混合起来：既有卷积，又有 LN/GELU/大通道 MLP。优化时不能只按老 ResNet 看。

## 6. Faster R-CNN

论文：Faster R-CNN, 2015。链接：https://arxiv.org/abs/1506.01497

为什么绝对经典：two-stage detection 的标准范式。很多高精度检测、实例分割、工业质检模型都从这条线演化。

核心机制：RPN 生成 proposals，RoI pooling/align 抽取候选区域特征，第二阶段做分类和 box regression。

工程含义：Faster R-CNN 不是单纯 backbone。瓶颈可能在 proposal 生成、RoIAlign、per-RoI head、小 tensor 操作和 NMS。DCU 优化时，如果只盯主干 conv，可能端到端没提升。

## 7. YOLO 系列

论文：You Only Look Once, 2015/2016。链接：https://arxiv.org/abs/1506.02640

为什么绝对经典：one-stage real-time detection 的主线。客户做实时检测、摄像头流、工业边缘部署，很可能用 YOLO 或其变体。

核心机制：单次前向直接预测 dense anchors/points 上的 box、class、objectness，后处理 decode + NMS。

工程含义：YOLO 的模型前向通常较规则，但后处理很关键：sigmoid、decode、top-k、NMS、坐标还原。如果 NMS 在 CPU 或 Python 上跑，模型 kernel 再快也会被端到端延迟吞掉。

优化重点：输入尺寸固定化、batching、conv fusion、INT8/BF16、NMS 下沉到设备侧、避免频繁 host-device 同步。

## 8. FPN

论文：Feature Pyramid Networks, 2016/2017。链接：https://arxiv.org/abs/1612.03144

为什么必须懂：检测和分割离不开多尺度。小目标、远处目标、细长目标都依赖高分辨率特征。

核心机制：自顶向下把高层语义传回低层分辨率，横向连接融合多尺度特征。

工程含义：FPN 带来大量不同 shape 的 feature map 和小卷积。优化时要注意多尺度分支的 kernel launch 开销、内存复用和 layout 转换。

## 9. Mask R-CNN

论文：Mask R-CNN, 2017。链接：https://arxiv.org/abs/1703.06870

为什么必须懂：实例分割经典基线。它把检测和 mask 分支统一起来，影响了后续大量实例级视觉任务。

核心机制：在 Faster R-CNN 上加 mask head，并用 RoIAlign 保持像素对齐。

工程含义：Mask R-CNN 端到端瓶颈经常在 RoIAlign、mask head、后处理和变长 proposal。它不像分类模型那样一个大 batch 规整前向就完事。

## 10. U-Net

论文：U-Net, 2015。链接：https://arxiv.org/abs/1505.04597

为什么必须懂：语义分割和医学分割的基础结构，也影响扩散模型中的 U-Net backbone。

核心机制：encoder 压缩语义，decoder 恢复分辨率，skip connection 补回细节。

工程含义：分割模型通常高分辨率特征多，显存压力大；upsample、concat、deconv、large feature map 是常见瓶颈。训练时 crop size、batch size、mixed precision 对显存影响非常明显。

## 11. DeepLab

代表：DeepLabv3/v3+。链接：https://arxiv.org/abs/1706.05587、https://arxiv.org/abs/1802.02611

为什么必须懂：语义分割主流路线之一，尤其是空洞卷积和 ASPP 对理解 dense prediction 很重要。

核心机制：空洞卷积扩大感受野但不降低分辨率；ASPP 用多尺度 dilation 捕获上下文。

工程含义：dilated conv 访问模式更稀疏，性能未必等同普通 conv。高分辨率 segmentation 上，显存带宽和 feature map 大小很关键。

## 12. DETR

论文：End-to-End Object Detection with Transformers, 2020。链接：https://arxiv.org/abs/2005.12872

为什么必须懂：DETR 把检测从 anchor/NMS 范式推向 set prediction + object query。后来的 Deformable DETR、DINO、BEVFormer、很多智驾 query-based 模型都受它影响。

核心机制：object queries 通过 Transformer decoder 从图像特征中提取目标信息，用 Hungarian matching 做集合预测。

工程含义：DETR 类模型引入 attention、query、matching。训练阶段 matching 在 CPU/Python 上也可能影响吞吐；推理阶段 query 数固定，后处理可能比 YOLO 少，但 attention 计算更重。

## 13. Deformable DETR

论文：Deformable DETR, 2020。链接：https://arxiv.org/abs/2010.04159

为什么必须懂：原始 DETR 收敛慢、小目标弱。Deformable attention 只在少量采样点上聚合多尺度特征，成为很多检测和智驾 BEV 方法的基础组件。

核心机制：每个 query 不 attend 全图，而是在参考点附近采样若干 key points，多尺度聚合。

工程含义：deformable attention 涉及采样、插值、scatter/gather，自定义算子多。适配 DCU 时，这是高风险区域：如果 fallback 到慢实现，性能会明显掉。

## 14. ViT

论文：An Image is Worth 16x16 Words, 2020。链接：https://arxiv.org/abs/2010.11929

为什么必须懂：ViT 把图像切成 patch 当 token，用 Transformer 做视觉表征，是 CV foundation model 的核心入口。

核心机制：patch embedding + position embedding + Transformer encoder。

工程含义：ViT 从卷积密集变成 GEMM/attention/LN 密集。高分辨率下 token 数按面积增长，attention 成本很快变大。优化时要关注 patch size、image resolution、window attention、FlashAttention 是否可用。

## 15. Swin Transformer

论文：Swin Transformer, 2021。链接：https://arxiv.org/abs/2103.14030

为什么必须懂：Swin 把 Transformer 做成层级化视觉 backbone，用 window attention 控制复杂度，并适配检测/分割。

核心机制：局部窗口 attention + shifted window 交互，不同 stage 逐步降分辨率升通道。

工程含义：window partition、shift、reshape、permute 可能带来额外开销。看起来是 Transformer，实际瓶颈可能在 layout 变换和小矩阵 attention。

## 16. MAE / DINOv2

代表：MAE, 2021；DINOv2, 2023。链接：https://arxiv.org/abs/2111.06377、https://arxiv.org/abs/2304.07193

为什么必须懂：视觉自监督是客户拿无标注数据做预训练/迁移的重要路线。DINOv2 代表大规模通用视觉特征。

核心机制：MAE 遮住大量 patch 做重建；DINO/DINOv2 用 teacher-student/self-distillation 学通用特征。

工程含义：预训练和下游推理 workload 不同。MAE 训练可能 encoder 只看可见 patch，decoder 轻量；DINO 类训练有 teacher/student 两套前向和多 crop 数据增强，数据 pipeline 和多视图 batch 很重。

## 17. CLIP

论文：Learning Transferable Visual Models From Natural Language Supervision, 2021。链接：https://arxiv.org/abs/2103.00020

为什么绝对经典：CLIP 让图像和文本进入共同语义空间，是 open-vocabulary、图文检索、多模态模型的基础。

核心机制：image encoder 和 text encoder 做对比学习，正确图文对相似度高，错误配对低。

工程含义：CLIP 系统不是单个 CV 模型。它有 image tower、text tower、projection、normalize、similarity matrix。训练时全局 batch 越大，对比学习越有效，但也带来跨卡 gather/通信成本。

客户场景：图文检索、开放类别分类、zero-shot 分类、VLM 视觉塔、RAG 图片检索，都可能用 CLIP/OpenCLIP。

## 18. SAM

论文：Segment Anything, 2023。链接：https://arxiv.org/abs/2304.02643

为什么必须懂：SAM 把分割做成 promptable foundation model，是通用分割能力的代表。

核心机制：重 image encoder 预计算图像特征，prompt encoder 编码点/框/mask，mask decoder 快速输出多个候选 mask。

工程含义：SAM 的优化要分阶段：image encoder 很重但可缓存；prompt decoder 轻但可能高频交互。端到端服务要看是否重复编码图片、是否批量处理 prompt、mask 后处理在哪跑。

## 19. Diffusion / Stable Diffusion

代表：DDPM, 2020；Latent Diffusion, 2021/2022。链接：https://arxiv.org/abs/2006.11239、https://arxiv.org/abs/2112.10752

为什么要知道：很多客户 CV 需求已经不是识别，而是生成、编辑、超分、控制生成。Stable Diffusion 类模型是典型工作负载。

核心机制：逐步去噪，常见结构是 U-Net + attention + VAE + text encoder。Latent Diffusion 在 latent 空间去噪，降低像素空间成本。

工程含义：扩散推理是多步迭代，单步模型不慢也可能总延迟高。瓶颈包括 U-Net attention、conv、scheduler 步数、VAE decode、ControlNet 多分支。优化要看 step 数、batch、分辨率和是否使用 fused attention/conv。

## 20. 多模态 VLM：BLIP / Flamingo / LLaVA

代表链接：https://arxiv.org/abs/2201.12086、https://arxiv.org/abs/2204.14198、https://arxiv.org/abs/2304.08485

为什么必须懂：客户可能拿“视觉模型”做图像问答、文档理解、GUI agent、视频理解。此时 CV 只是视觉塔，后面还有 LLM。

核心机制：视觉 encoder 提特征，通过 projector/resampler 接入 LLM，上层用语言模型完成推理和生成。

工程含义：VLM 是混合 workload：图像 encoder 是 CV，LLM decode 是 NLP。优化时要拆开测：vision prefill、multimodal projector、LLM prefill/decode、图像 token 数。很多 VLM 慢在视觉 token 太多导致 LLM 上下文膨胀。

## 21. 智驾相关 CV：BEVFormer / BEVDepth / PETR

这些已经在《智驾经典研究与技术脉络》里单独整理，但 CV 基础里也必须知道它们的位置：它们不是普通 2D CV，而是多相机 3D 感知，把图像特征转换到 BEV 或 3D query 空间。

工程含义：智驾 CV 常见瓶颈包括多相机 batch、view transform、grid sample、deformable attention、时序 memory、NMS/track、occupancy head。DCU 适配时要特别小心自定义 CUDA 算子迁移问题。

## 面向 DCU 优化的判断清单

拿到客户 CV 模型，先问：

1. 任务是什么：分类、检测、分割、实例分割、生成、VLM、智驾 BEV？
2. backbone 是什么：ResNet、MobileNet、ConvNeXt、ViT、Swin、CLIP vision tower？
3. 输入形状：固定分辨率还是动态？单图还是多相机/视频？batch 多大？
4. 主要算子：conv、depthwise conv、attention、deformable attention、RoIAlign、grid_sample、upsample、NMS？
5. 后处理在哪：CPU、Python、设备端？有没有 host-device sync？
6. 精度策略：FP32、FP16、BF16、INT8？BN/LN/softmax/resize 是否有精度坑？
7. 端到端指标：不是只看 model forward，还要看 preprocess、postprocess、吞吐、延迟、显存和精度。

真正合格的 CV 优化是能解释：这个任务为什么需要这种结构，结构里哪些算子决定速度，优化后是否保持检测框、mask、分割边界、开放词表或生成质量。

## 阅读路线

做通用 CV 优化：ResNet -> FPN -> Faster R-CNN/YOLO -> Mask R-CNN/U-Net -> DETR/Deformable DETR -> ViT/Swin。

做视觉基础模型：ViT -> MAE/DINOv2 -> CLIP -> SAM -> VLM。

做智驾：先补通检测/分割/ViT/DETR，再读 LSS、BEVDet、BEVDepth、PETR、BEVFormer、MapTR、SurroundOcc、UniAD。
