自动混合精度（Automatic Mixed Precision, AMP）最核心的价值，是在尽量不改模型逻辑的前提下，用更低精度完成一部分计算，从而降低显存占用、提升训练吞吐。

它不是“把所有东西都换成 fp16”这么简单。真正稳定可用的 AMP，一般都依赖框架帮你决定哪些算子适合低精度、哪些算子必须留在更高精度下执行。

## 为什么要用 AMP

- 更低的显存占用：同样的 batch size 下，更容易塞进显存
- 更高的吞吐：Tensor Core 对半精度/混合精度通常更友好
- 更灵活的训练配置：同一块卡上能尝试更大的模型、更长的序列或更大的 batch

在现代 GPU 上，AMP 往往是默认应当尝试打开的优化项。只有当模型特别小、瓶颈不在矩阵计算、或者数值非常脆弱时，它的收益才可能不明显。

## 两个核心组件

PyTorch 里最常见的 AMP 组合是：

- `torch.autocast`：控制前向和部分算子以合适的低精度运行
- `torch.amp.GradScaler`：对 loss 做缩放，避免反向传播时梯度下溢

可以把它们理解成：

- `autocast` 负责“该用什么精度算”
- `GradScaler` 负责“低精度反传时别让梯度太小直接变成 0”

## 最小训练示例

```python
import torch

model = model.cuda()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)
scaler = torch.amp.GradScaler("cuda")

for x, y in dataloader:
    x = x.cuda(non_blocking=True)
    y = y.cuda(non_blocking=True)
    optimizer.zero_grad(set_to_none=True)

    with torch.autocast(device_type="cuda", dtype=torch.float16):
        logits = model(x)
        loss = criterion(logits, y)

    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

如果你已经在用 `bf16`，很多场景下也可以把 `dtype` 改成 `torch.bfloat16`。`bf16` 的动态范围更大，训练时通常比 `fp16` 更稳，但具体收益要看硬件支持。

## 什么时候需要 `GradScaler`

- `fp16` 训练：通常需要
- `bf16` 训练：很多时候不需要，因为 `bf16` 的指数范围更大

一个常见误区是“只要开了 autocast 就够了”。如果你用的是 `fp16`，没有 loss scaling 时，梯度很容易因为数值太小而下溢，表现就是 loss 不动、梯度全 0、模型完全学不起来。

## 常见问题

### 1. Loss 变成 NaN

通常先检查：

- 学习率是不是本来就过大
- loss 里有没有非常激进的 `exp`、`log`、除法
- 输入数据里是否有异常值
- 是否把一些本应保留高精度的计算强行转成了 `half`

如果模型在纯 `fp32` 下也不稳定，那就不是 AMP 的锅，只是 AMP 更早把问题暴露出来。

### 2. 速度没有明显提升

这并不罕见。可能原因包括：

- 模型太小，GPU 根本没吃满
- 瓶颈在数据加载、CPU 预处理、通信或 I/O
- 当前硬件对低精度加速不明显
- 算子类型本身不太受益于 Tensor Core

所以 AMP 是高概率收益项，但不是机械地“打开就一定快很多”。

### 3. 某些算子报精度或类型错误

这通常说明：

- 某个自定义算子没有正确处理半精度
- 代码里手动写死了 dtype
- 某些中间张量被错误地强转了

这种时候不要急着全局关 AMP，先定位是哪一段算子不兼容，再局部回退到 `fp32` 往往更合理。

## 实践建议

- 优先使用框架原生 AMP 写法，不要手搓全局 `.half()`
- 把 AMP 当成训练配置的一部分，和 batch size、学习率一起看
- 遇到数值问题，先做 A/B：`fp32` 能不能稳定，再比较 `fp16` / `bf16`
- 若是大模型训练，优先关注 `bf16` 是否可用

## 总结

AMP 的本质不是“偷精度”，而是把数值稳定性和训练效率做工程化平衡。真正有用的不是记住几个 API，而是知道哪些收益来自低精度、哪些风险来自梯度和数值范围，以及出问题时该先查模型、算子还是硬件。
