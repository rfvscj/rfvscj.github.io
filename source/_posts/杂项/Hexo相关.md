这篇不是系统教程，而是几个实际维护博客时容易踩坑的点。

## Hexo + Obsidian + Git

这套组合适合长期写博客，但要注意：Obsidian 是写作和整理区，Hexo 是发布区，不要把所有零散笔记直接同步成公开文章。

## 文档加密

如果要给单篇文章加密码，可以使用 `hexo-blog-encrypt`，然后在 front matter 里添加：

```yml
password: your-password
```

这种方式适合少量需要限制访问的内容，不适合把大量常规文章都做成加密页。

## 隐藏文章

这个仓库现在的统一做法是：

```yml
published: false
```

不再使用 `notshow`，也不再新增 `hide`。原因很简单：`published: false` 更清晰，也更不依赖主题实现细节。

## 站内链接

如果要稳定地引用另一篇文章，优先使用：

```md
{% post_link 文章文件名 '显示文字' %}
```

这里一般写文章文件名本身，不带目录路径。这样即使 permalink 改了，站内链接也不容易失效。

## 一个维护建议

如果某篇文章只是临时记录的命令、格式或一两句备忘，不要急着发出去。先判断它是：

- 能独立阅读的文章
- 可合并进别的文章的片段
- 只该留在本地草稿里的笔记

博客不是笔记仓库的原样外露，发布前多做一步整理，整体质量会好很多。
