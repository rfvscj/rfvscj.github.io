简单来讲，尾递归是指在一个方法内部，递归调用后直接 `return`，没有任何多余指令了。

尾递归之所以重要，不是因为“看起来高级”，而是它在一些支持尾调用优化的语言或编译器里，可以避免不断增长调用栈。

## 一个不是尾递归的例子

例如简单的累加函数：

```python
def acc(n):
	if n == 1:
		return 1
	return n + acc(n - 1)
```

以上代码不是尾递归，因为计算完 `acc(n - 1)` 后，还要再做一步 `+ n`。

## 尾递归写法

```python
def acc(n, sum=0):  # 函数意义：sum + n的累加
	if n == 1:
		return sum + n  # n==1时，sum为1后边的和
	return acc(n - 1, sum + n)  # n>1，sum加上当前的n作为n及之后的所有和
```

这里递归调用已经是最后一步了，返回值直接就是下一层的返回值，这才符合尾递归的定义。

## 为什么它可能更高效

如果语言或编译器支持尾调用优化，那么尾递归调用时可以复用当前栈帧，而不是每深入一层都额外压栈。

但这里要注意一件事：

- “尾递归更高效”不是普适真理
- 它依赖语言、编译器和运行时是否真的做了优化

例如 Python 默认并不会因为你写成尾递归就自动做尾调用优化，所以在 Python 里，尾递归更多是一个概念上的区分，而不是性能技巧。

## 总结

尾递归的判断标准很明确：递归调用之后，不能再有额外计算。至于它会不会真的更省栈、更快，要看具体语言是否支持尾调用优化。
