# 导读

这个博客不会维护成一个固定目录。它更像一个逐步生长的知识库：先把工作中遇到的问题写下来，再把反复出现的主题沉淀成更稳定的文章。

所以这个页面只给阅读方向，不列具体文章清单。真正的文章入口看分类页和归档页。

## 技术主线

### LLM / NLP

大模型、传统 NLP、训练目标、推理部署、RAG、Agent、多模态和性能优化都会放在这里。这个栏目更偏“理解客户模型在做什么”，不是单纯追论文标题。

入口：[LLM必备](/categories/LLM必备/)

### CV / 智驾

视觉模型、检测、分割、BEV、时序融合、occupancy、规划和端到端智驾相关内容会放在这里。这个栏目会比过去更重要，因为它和当前工作场景更贴近。

入口：[CV必备](/categories/CV必备/)

### GPU / DCU / 系统

算力平台、混合精度、算子、推理训练性能、网络、部署和工具链问题会放在这里。后续这个方向应该逐步从零散经验变成更系统的工程笔记。

入口：[GPU相关](/categories/GPU相关/)

### OS / 网络 / 工作流

Linux、Git、Hexo、Clash、Obsidian、远程开发和日常工具链问题会放在这里。它们不一定宏大，但会反复用到。

入口：[OS&网络](/categories/OS-网络/)

## 非技术主线

### 思考

行业观察、阶段复盘、个人选择和长期判断放在这里。这部分不会追求更新频率，只保留有必要公开沉淀的内容。

入口：[思考](/categories/思考/)

### 杂项

暂时没有归入主线的算法、基础概念和旧笔记会留在这里。后续如果某类内容变多，再拆出新的稳定栏目。

入口：[杂项](/categories/杂项/)

## 阅读方式

- 想按主题读：从[专题](/categories/)进入。
- 想看最近写了什么：从[文章](/archives/)进入。
- 想了解这个博客为什么这样组织：看[关于](/about/)。
- 想维护这个仓库：看[站点维护](/guide/)。

这个页面只维护方向，不维护具体文章列表。文章之间有明确依赖时，再在正文里互相引用。
